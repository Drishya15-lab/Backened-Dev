# Authentication Practice Problems

This repository contains solutions to 6 practice problems for building authentication and authorization systems in Express.js.

## Problems

### Problem 1: Secure User Registration System
- File: `problem1.js`
- Features: Password validation, bcrypt hashing, duplicate prevention
- Run: `node problem1.js`

### Problem 2: Session-Based Shopping Cart
- File: `problem2.js`
- Features: Session persistence, cart operations (add, update, remove, get)
- Run: `node problem2.js`

### Problem 3: Multi-Role Authorization System
- File: `problem3.js`
- Features: Role-based access control (user, moderator, admin), ownership checks
- Run: `node problem3.js`

### Problem 4: JWT Authentication with Refresh Tokens
- File: `problem4.js`
- Features: JWT access/refresh tokens, token refresh, logout
- Run: `node problem4.js`

### Problem 5: Passport.js with Multiple Strategies
- File: `problem5.js`
- Features: Local and JWT strategies with Passport.js
- Run: `node problem5.js`

### Problem 6: Rate-Limited Login with Account Lockout
- File: `problem6.js`
- Features: Login rate limiting, account lockout after failed attempts
- Run: `node problem6.js`

## Setup

1. Install dependencies: `npm install`
2. Run any problem: `node problemX.js`

Note: Each problem runs on port 3000, so run them separately.

## Test Cases

Each problem includes test cases as described in the original requirements.
